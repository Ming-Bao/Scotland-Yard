Detailed scores
Criterion Google Maps Leaflet MapLibre GL
Performance
Render speed measured via Firefox headless — 3 runs each, navigation start → "Done."
3
Slowest at baseline (0.45 s mean) with high first-run variance (up to 1.13 s from API initialisation overhead). Ties Leaflet at the 200-node stress scale (both 2.33 s mean). [11]
4
Fastest at baseline (0.19 s mean). Matches Google Maps at the 200-node stress scale (2.33 s mean) and is only 0.25 s behind MapLibre GL. DOM rendering does not limit performance at Scotland Yard game scale. [11]
5
Most consistent across both scales. Best at the 200-node stress test (2.08 s mean, lower variance than either alternative). WebGL initialisation adds ~0.11 s at baseline but pays off under load. [11]
Cost
Licensing and usage fees at game scale
1
~$7 USD per 1,000 Dynamic Map loads after a $200/month free tier. Requires a billing account and active API key for any deployment. [1]
5
Open source — no library fees. CartoDB tiles used in the demo are free for non-commercial and low-traffic use cases.
5
Open source — no library fees. Supports the same free tile providers as Leaflet, including CartoDB and any self-hosted tile server.
License
Usage rights, open source, offline capability
3
Proprietary ToS — tile caching and self-hosting are prohibited. For this non-commercial university project the offline and redistribution restrictions are not a practical concern. [2]
5
BSD-2-Clause. Highly permissive — commercial use, modification, and redistribution all allowed without restriction. [3]
5
BSD-3-Clause. Open-source community fork of Mapbox GL JS v1.13, created after Mapbox switched to a proprietary licence in December 2020. Fully permissive for all use cases. [6][9][10]
Usability & Visual
End-user map quality, visual polish, tile rendering
5
Most polished and user-familiar basemap. Custom JSON styling produces a clean dark game aesthetic with road hierarchy, water, and POI control. Professional-grade out of the box.
3
Visual quality depends entirely on the chosen tile provider. Default CartoDB tiles produce a functional but dated appearance. Raster tiles lack the resolution-independent zoom of vector rendering.
4
Smooth, resolution-independent vector tile rendering with crisp zoom transitions. High-quality dark-mode styling is achievable via the GL Style Spec. Slightly less familiar to end-users than Google Maps.
Customisation
Styling flexibility, plugin ecosystem, renderer control
3
JSON styling API allows colour and visibility changes, but is locked to Google's platform. Custom tile servers and rendering overrides are not possible.
4
Large plugin ecosystem (551+). Supports custom tile layers, GeoJSON overlays, and divIcon markers. Flexible for most scenarios but bounded by DOM rendering. [4]
5
Full GL Style Specification — complete control over sprites, fonts, expressions, and data sources. Best-in-class flexibility for a fully custom game map. [8]
Library Quality
Active maintenance, committer diversity, issue handling
5
Backed by Google with a large dedicated engineering team. Actively maintained, extensive documentation, and a very stable versioned API.
3
The v1.x branch is in maintenance mode (critical fixes only) and v2.0.0-alpha remains pre-release. Small core team; some issues have remained open for years. [5]
4
Active open-source community with multiple corporate and individual committers. Issues are handled with regular releases and a growing ecosystem. [7]
